| **LLM01: Prompt Injection**                 | Prompt Injection occurs when an attacker manipulates a Large Language Model (LLM) through crafted inputs, causing the model to execute the attacker's intentions unintentionally. | **LLM02: Sensitive Information Disclosure** | Sensitive Information Disclosure in LLMs refers to the unintended leakage of confidential data, such as personal identification information (PII), financial details, or proprietary algorithms, through the model's outputs. This vulnerability arises when models are trained on or have access to sensitive data without adequate safeguards. |                                                                                 |
| **LLM03: Supply Chain**                     | The LLM supply chain is vulnerable to various risks due to the complexity and opacity of the training data, model weights, and deployment environments. Compromised components, such as poisoned datasets or tampered pre-trained models, can introduce security flaws that are difficult to detect and mitigate. |                                                                                 |
| **LLM04: Data and Model Poisoning**         | Data and Model Poisoning refers to the manipulation of training data or model parameters to introduce vulnerabilities, backdoors, or biases that compromise the model's security, effectiveness, or ethical behavior. This can result in the model producing incorrect outputs or behaving maliciously under specific conditions. |                                                                                 |
| **LLM05: Improper Output Handling**         | Improper Output Handling refers to the insufficient validation, sanitization, and handling of the outputs generated by LLMs before they are passed downstream to other systems or users. This vulnerability can lead to security issues like Cross-Site Scripting (XSS) or Server-Side Request Forgery (SSRF) if the content is rendered directly without scrutiny. |                                                                                 |
| **LLM06: Excessive Agency**                 | Excessive Agency refers to the vulnerability caused by an LLM system having too much autonomy or authority to perform actions, such as calling APIs or executing functions, without proper constraints. This can lead to unintended or harmful consequences if the model hallucinates or is manipulated into performing unauthorized operations. | Least privilege in tooling, access credentials, network and file system access. |
| **LLM07: System Prompt Leakage**            | System Prompt Leakage occurs when the system prompts or instructions that define the LLM's behavior and constraints are inadvertently revealed to users through the model's outputs. This exposure can provide attackers with insights into the system's logic, facilitating targeted attacks or prompt injection exploits. |                                                                                 |
| **LLM08: Vector and Embedding Weaknesses**  | Vector and Embedding Weaknesses arise in systems utilizing Retrieval-Augmented Generation (RAG) where the integrity and security of the vector database or the embeddings themselves are compromised. This can lead to the retrieval of unauthorized data or the injection of malicious content into the context window. |                                                                                 |
| **LLM09: Misinformation**                   | Misinformation, often referred to as hallucinations, occurs when an LLM generates false, misleading, or nonsensical information that is presented as fact. This can lead to the spread of inaccurate information, reputational damage, and poor decision-making if users rely on the model's output without verification. |                                                                                 |
| **LLM10: Unbounded Consumption**            | Unbounded Consumption is a vulnerability where a user is able to make an excessive number of requests to an LLM, leading to resource exhaustion, denial of service, or significant financial loss. This occurs when rate limiting or resource allocation controls are not properly implemented or enforced. |                                                                                 |
